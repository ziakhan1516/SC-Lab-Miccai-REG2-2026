# Algorithm predictions (local test)

> ⚠️ **Bundled files are placeholders.** The prediction files shipped here contain dummy answers (`"Placeholder answer for local test."`) and chain-of-thought steps copied from the bundled training-set ground truth. They exist only to let you run the pipeline end-to-end on the sample cases. Replace them with outputs from your own algorithm.

Provide **at least one** file below, then run [`../../prepare_test_input.sh`](../../prepare_test_input.sh) from the package root (after `./prepare_test_ground_truth.sh`).

Each `id` must match a case from [`../cases/`](../cases/) via `ground_truth/manifest.json`. See [`../README.md`](../README.md#linking-cases-predictions-and-manifest) for the full ID-linking rules.

Missing files are treated as **no predictions** for that interface. A file may also be an empty array `[]`.

---

## interf0 — `interf0/predictions.json`

One entry per ROI. The `id` must match the `anonymous_id` in `cases/interf0/rois_mapping.txt` (file extension stripped if present).

```json
[
  { "id": "000000", "answer": "No tissue is visible in this background ROI." },
  { "id": "000001", "answer": "Yes, tissue is clearly visible in this ROI." }
]
```

- `id` must also appear in `ground_truth/manifest.json` (generated by step 1).
- `answer` is a plain text string — the model's response to the question in `rois_mapping.txt`.

---

## interf1 — `interf1/predictions.json`

One entry per WSI. The `id` must match a case `id` in `cases/interf1/ground_truth_CoT.json` (file extension optional).

```json
[
  {
    "id": "PIT_01_00020_01.tiff",
    "chain-of-thought": [
      { "question": "What is the organ?", "answer": "Breast", "next_question": "Is there any abnormality present?" },
      { "question": "Is there any abnormality present?", "answer": "Yes, there is an abnormality.", "next_question": "..." },
      { "question": "What is the final pathology report?", "answer": "Breast, core needle biopsy; ...", "next_question": "" }
    ]
  }
]
```

- The last step must have `"next_question": ""`.
- Do **not** wrap the array in a `{"id": ..., "chain-of-thought": ...}` top-level object — the file must be a bare JSON array.

---

## How predictions flow through the pipeline

```
data/predictions/interf0/predictions.json
data/predictions/interf1/predictions.json
          │
          ▼  prepare_test_input.sh
test/input/predictions.json              ← job list read by the Docker container
test/input/<pk>/output/visual-context-response.json   ← interf0 answer
test/input/<pk>/output/chain-of-thought.json          ← interf1 steps
          │
          ▼  do_test_run.sh
test/output/metrics.json                 ← aggregated scores
```

Run [`../../do_test_run.sh`](../../do_test_run.sh) after the prep scripts; results go to [`../../test/output/metrics.json`](../../test/output/metrics.json).
